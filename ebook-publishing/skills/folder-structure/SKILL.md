# Folder Structure Enforcement

Audits the book or series directory against the expected layout, warns about misplaced files, and offers to move them with user confirmation. Never deletes files.

## Expected Layout

**Standalone book** (rooted at the book directory):

```
<book-root>/
  AGENTS.md         ← agent orientation file (auto-generated by this skill)
  CLAUDE.md         ← Claude Code project instructions (auto-generated by this skill)
  book-profile.md
  planning/         ← beat sheets, outlines, character notes, world-building
  drafts/           ← chapter files (chapter-01.md, ch-01.md, etc.)
  editing/          ← edited chapters, editor feedback docs
  publishing/       ← blurbs, metadata, formatted output (epub/mobi/pdf)
  covers/           ← draft cover images and cover-brief.md
  covers/cover-final.{ext}  ← approved final cover image (single file, named exactly)
  covers/prompts/   ← AI cover generation prompt files
```

**Series** (rooted at the series directory):

```
<series-root>/
  AGENTS.md         ← agent orientation file (auto-generated by this skill)
  CLAUDE.md         ← Claude Code project instructions (auto-generated by this skill)
  series-profile.md       ← optional
  book-01/
    book-profile.md
    planning/
    drafts/
    editing/
    publishing/
    covers/
    covers/prompts/
  book-02/ ...
```

## Step 1 — Detect layout type

Read `book-profile.md` if present. Check the `series` frontmatter field:
- `series: null` → standalone book layout
- `series: "<name>"` → series layout; scan for `book-NN/` subdirectories

If no `book-profile.md` exists, scan for `book-NN/` subdirectories to infer series layout. If still ambiguous, ask the user before proceeding.

## Step 2 — Scan and classify

Walk the full directory tree. Classify every file:

| Status | Meaning |
|--------|---------|
| **Expected** | Correct location for its type — skip |
| **Misplaced** | Wrong subdirectory — report with suggested destination |
| **Unknown** | Doesn't match known patterns — ask user where it belongs |

**File-to-folder mapping:**

| File pattern | Belongs in |
|---|---|
| `AGENTS.md` | series root (series) or book root (standalone) |
| `CLAUDE.md` | series root (series) or book root (standalone) |
| `book-profile.md` | book root |
| `series-profile.md` | series root |
| Beat sheets, outlines, story structure `.md/.txt` | `planning/` |
| Character sheets, world-building, research notes | `planning/` |
| `chapter-*.md`, `ch-*.md`, prose draft files | `drafts/` |
| Edited chapter files, editor feedback `.md/.txt` | `editing/` |
| Blurbs, descriptions, keywords, metadata `.md/.txt` | `publishing/` |
| Formatted output: `.epub`, `.mobi`, `.pdf` | `publishing/` |
| `.docx` files (read-only, from external tools) | `publishing/` |
| `cover-final.png` / `cover-final.jpg` / `cover-final.webp` | `covers/` — this is the approved final cover; only one should exist |
| `cover-brief.md` | `covers/` |
| Draft cover images (any other `.png`, `.jpg`, `.jpeg`, `.webp`) | `covers/` |
| Cover prompt `.md/.txt` files | `covers/prompts/` |

## Step 3 — Report findings

Present a summary before taking any action:

```
Folder structure audit — <book title or directory name>

Expected folders:
  planning/       ✓
  drafts/         ✓
  editing/        ✗ (missing)
  publishing/     ✗ (missing)
  covers/         ✓
  covers/prompts/ ✓

Misplaced files:
  chapter-01.md      (root)      → drafts/
  my-outline.md      (root)      → planning/
  cover-draft.png    (drafts/)   → covers/

Unknown files (destination unclear):
  notes-from-editor.docx  (root)
  old-version.txt          (planning/)
```

If no issues found, confirm structure is clean and stop.

## Step 4 — Offer actions

Present options:

> "For misplaced files I can:
> a) Move all to their suggested locations
> b) Decide case-by-case
> c) Move some or all outside the book folder (specify destination)
> d) Leave everything as-is
>
> For unknown files I'll ask you individually where each should go."

Wait for user response before moving anything.

## Step 5 — Execute moves

- Move only files the user approved
- Create destination directories if they don't exist (report each one created)
- Report each move: `Moved: chapter-01.md → drafts/chapter-01.md`
- If a file already exists at the destination, warn and skip — never overwrite

## Step 6 — Create or update AGENTS.md

After completing the audit, write `AGENTS.md` at the project root (series root for a series, book root for a standalone). If it already exists, update it in place.

Read `book-profile.md` and `series-planner.md` (if present) to populate the project-specific sections.

```markdown
# [Series Name / Book Title] — Agent Orientation

## What this project is

[For series:]
This is a multi-book fiction series titled **[Series Name]**. It contains [N] planned books.
Refer to `series-planner.md` for the full series arc, per-book summaries, and character trajectories.
Refer to each `book-NN/book-profile.md` for per-book metadata.

[For standalone:]
This is a standalone fiction book titled **[Book Title]** by [Author].
Refer to `book-profile.md` for full metadata: genre, protagonist, core conflict, target audience, and locale.

## Project summary

[Pull 2–3 sentences from the book-profile.md Book Summary, or from series-planner.md series summary if series.]

## Folder structure

### [For series: series root | For standalone: book root]

| Path | Purpose |
|---|---|
| `AGENTS.md` | This file — agent orientation |
| `book-profile.md` | Book metadata: title, genre, protagonist, conflict, locale, comps |
| `series-planner.md` | Series arc, per-book summaries, character trajectories (series only) |
| `planning/` | Beat sheets, outlines, character profiles, world-building, writing style guide |
| `drafts/` | Chapter files in progress (`chapter-01.md`, etc.) |
| `editing/` | Edited chapters, structural edit reports, line-edit outputs, proofread outputs |
| `publishing/` | Book descriptions, publishing metadata, marketing copy, formatted output files |
| `covers/` | Draft cover images, `cover-brief.md`, and `cover-final.*` (approved final cover) |
| `covers/prompts/` | AI image generation prompt files for cover concepts |

[For series, also include:]
| `book-01/` … `book-NN/` | Per-book directories, each containing the structure above |

```

Also write `CLAUDE.md` at the same root. If it already exists, update it in place. This file is read by Claude Code at session start and by the built-in `/init` command — it makes the ebook-publishing context self-propagating without any manual setup.

```markdown
# [Series Name / Book Title]

This is an **ebook-publishing project** managed with the [ebook-publishing plugin](https://github.com/your-repo) for Claude Code.

## On every session start

Invoke the `folder-structure` skill immediately. It will:
- Audit the directory layout and flag misplaced files
- Keep `AGENTS.md` and this file up to date
- Ensure expected directories exist

## Project context

See `AGENTS.md` for full project orientation: what this series/book is, the folder structure reference, and the complete skill inventory.

## Key files

[For series:]
- `series-planner.md` — series arc and per-book structure
- `book-NN/book-profile.md` — per-book metadata

[For standalone:]
- `book-profile.md` — title, genre, protagonist, conflict, locale, comps

## Conventions

- All Claude-generated output files use `.md` format
- `.docx` files are read-only inputs from external tools — never create or write them
- Chapter drafts belong in `drafts/`, edited versions in `editing/`
- Final approved cover image is always named `covers/cover-final.*`
```

## Core rules

1. **Never create or write `.docx` files** — `.docx` is read-only input from external tools (e.g., Word, Scrivener). All plugin output uses `.md`.
2. **Never delete** — only move, and only with explicit user approval
3. **Report first, act second** — show full audit before touching anything
4. **Create missing folders** — if `planning/` doesn't exist, create it before moving files into it
5. **Unknown files always ask** — never assume a destination for files that don't match known patterns
6. **Run proactively** — other ebook-publishing skills should invoke this if they encounter misplaced files or missing expected directories
